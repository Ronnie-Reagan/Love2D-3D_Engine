# Long-Term Model Module Plan and Full System Review

## Summary
Design and document a durable `model_module` architecture that can support `.glb`, `.gltf`, `.stl`, full PBR, 2048 paint overlays, multiplayer sync, and restart persistence without continuing to expand `main.lua` coupling.  
This plan is documentation-first and review-first, with explicit changes to the earlier feature plan to improve long-term maintainability.

## Implications and Risks vs Current Design

1. Expanding directly in `main.lua` will accelerate technical debt and make future format/material additions high-risk.  
2. Full CPU parity for PBR + paint at 2048 is possible but will be expensive; without capability profiling, low-end systems will regress hard.  
3. 2048 overlays cost ~16MB each uncompressed; with plane + walking + peer variants, memory pressure will spike unless capped/lazy-loaded.  
4. `.gltf` sidecar handling introduces path-security risk; relative sandbox policy is mandatory.  
5. No backward network compatibility simplifies architecture but requires explicit protocol versioning and hard fail for older clients.  
6. Manual-only QA is insufficient for long-term module safety; minimum parser/network contract tests are needed even if full visual tests stay manual.

## Plan Changes for Long-Term Module Viability (Adopted)

1. Change from “feature-first edits” to “module-first architecture” before adding PBR/paint features.  
2. Replace model-specific transport with generic blob transport for both model and paint payloads.  
3. Add renderer capability profiles (`GPU_FULL`, `CPU_FULL`, `CPU_SAFE`) while preserving your full parity target as the primary path.  
4. Add minimal automated contract tests for loader/protocol correctness; keep visual validation manual smoke.  
5. Add architecture docs/ADR deliverables as first-class outputs.

## Target Module Architecture

1. `model_module/init.lua`: public API, orchestration, cache ownership.  
2. `model_module/importers/stl_importer.lua`: current STL behavior isolated.  
3. `model_module/importers/gltf_importer.lua`: GLB/GLTF parsing, accessors, textures, transforms.  
4. `model_module/json_decode.lua`: internal JSON decoder for glTF payloads.  
5. `model_module/material_runtime.lua`: canonical material representation and texture binding.  
6. `model_module/paint_runtime.lua`: overlay textures, brush ops, undo/redo, commit hash generation.  
7. `model_module/render_bridge_gpu.lua`: mesh/material upload and shader binding contract.  
8. `model_module/render_bridge_cpu.lua`: CPU material sampling/parity raster contract.  
9. `model_module/net_blob_sync.lua`: generic chunked transfer for model/paint blobs.  
10. `model_module/persistence.lua`: restart snapshot and disk persistence integration.  
11. `model_module/diagnostics.lua`: budgets, validation, and user-facing import diagnostics.

## Integration Plan With Existing Files

1. `main.lua`: replace direct STL model-load calls with `ModelModule` API; keep existing pause and render flow but route through module.  
2. `engine.lua`: keep math/raster utilities; remove direct ownership of future model format logic.  
3. `gpu_renderer.lua`: consume module-provided render batches/material bindings instead of building color-only mesh ad hoc.  
4. `networking.lua`: migrate state packet parsing to `STATE2` fields and integrate blob sync hooks.  
5. `controls.lua`: add paint actions for brush/erase/fill/undo/redo/commit mapped to pause editor usage.

## Revised Delivery Phases

1. **Phase 0: Documentation and ADRs**  
Deliver `docs/model-module-review.md`, `docs/model-module-spec.md`, `docs/model-module-migration.md`, `docs/adr/adr-0001-model-module.md`.  
Exit criteria: interfaces frozen and migration map approved.

2. **Phase 1: Module Scaffolding and STL Adapter**  
Create module skeleton and route existing STL import through adapter with zero behavior regression.  
Exit criteria: current STL gameplay unchanged, module API active.

3. **Phase 2: GLB/GLTF Importers (Core 2.0 Only)**  
Implement GLB chunk parser, GLTF accessor decoding, node transforms, relative sandbox URI policy, and extension rejection diagnostics.  
Exit criteria: `.glb` and `.gltf` path-load working; drag/drop policy enforced.

4. **Phase 3: Material Runtime and GPU PBR**  
Implement full PBR material ingestion and GPU shader/render batch path, including alpha mode handling.  
Exit criteria: textured PBR assets render correctly in GPU path.

5. **Phase 4: CPU Full-Parity Material Path**  
Implement CPU texture/material sampling and PBR shading parity path with `CPU_SAFE` fallback profile available.  
Exit criteria: CPU mode visually consistent within defined tolerance and remains interactive.

6. **Phase 5: Paint Runtime + Pause Editor**  
Add 2048 overlay system, brush/erase/fill, undo/redo, commit/revert, and preview/world application.  
Exit criteria: complete local paint workflow functional for plane and walking roles.

7. **Phase 6: Generic Blob Sync + Restart Persistence**  
Adopt `BLOB_*` protocol for model and paint payloads, integrate `STATE2`, and persist paint in restart snapshots.  
Exit criteria: multiplayer model+paint sync works and restart restores both.

8. **Phase 7: Hardening and Budget Enforcement**  
Apply strict caps, error messages, diagnostics, and memory eviction/lazy-load policy for overlays and textures.  
Exit criteria: predictable behavior under oversized or malformed assets.

## Runtime Budgets (Long-Term Safe Defaults)

1. Source file size: 100MB max.  
2. Max triangles per asset: 120,000.  
3. Max vertices per asset: 200,000.  
4. Max materials per asset: 24.  
5. Max textures per asset: 12.  
6. Max texture dimension: 4096.  
7. Max decoded texture memory per asset: 256MB.  
8. Paint overlay default: 2048x2048; downgrade to 1024 under memory pressure profile.  
9. Blob chunk size: keep current model chunk baseline and move to shared chunking policy across model/paint.

## Protocol Plan (No Backward Compatibility)

1. Replace legacy positional packet dependence with explicit `STATE2` key/value schema.  
2. Add protocol version token at connect/first state.  
3. Reject mismatched protocol peers immediately with clear status text.  
4. Use `BLOB_REQUEST`, `BLOB_META`, `BLOB_CHUNK` for both model and paint transfer.

## Test Cases and Scenarios

1. Import `.stl`, `.glb`, `.gltf` and validate expected object appearance and role assignment.  
2. Reject `.gltf` absolute/escaping URIs and unsupported compression extensions with clear errors.  
3. Validate GPU PBR and CPU parity on baseColor, normal, metallic/roughness, occlusion, emissive, alpha modes.  
4. Verify paint editor operations: stroke, erase, fill, undo, redo, commit, revert.  
5. Verify paint commit sync between two clients via blob transfer and hash validation.  
6. Verify restart restores model hash, material state, paint hash, and orientation per role.  
7. Validate memory cap behavior by loading high-texture assets and multiple paint overlays.  
8. Validate drag/drop policy: `.glb` accepted, `.gltf` with sidecars rejected in drop flow.  
9. Validate protocol mismatch handling between updated and old clients.

## Assumptions and Defaults

1. No license enforcement in-app, but importer logs source metadata when present.  
2. Full CPU parity remains target behavior; `CPU_SAFE` exists only as operational fallback.  
3. 2048 paint is default, with adaptive downgrade under memory pressure for stability.  
4. Manual per-model orientation controls are the canonical alignment path.  
5. Network backward compatibility is intentionally dropped and version-gated.  
6. Manual smoke remains primary validation, with added minimal automated contract tests for parser/protocol longevity.
